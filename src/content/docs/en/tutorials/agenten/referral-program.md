# Referral Program Designer

## When to use

- User wants to build or optimize a referral program
- User asks about viral growth strategy
- User wants to design incentives for word-of-mouth referrals

## The Referral Loop

```
Satisfied user → Shares invite → New user signs up
→ Both receive reward → New user becomes satisfied → Shares further
```

## Incentive Structure

| Model | Description | Example |
|-------|-------------|---------|
| One-sided | Only referrer gets reward | €20 credit per referral |
| Double-sided | Both parties receive reward | €10 for both |
| Tiered | More referrals = better rewards | Silver/Gold/Platinum |

**Reward types:**
- Account credit / discounts (simple, effective)
- Free upgrade (high perceived value)
- Cash (more compliance needed, high motivation)
- Donation / karma (niche market, trust-building)

## Trigger Moments

Best moments to request referrals:
- Directly after first success / aha moment
- After a positive support interaction
- After renewal or upgrade
- After receiving a key value (5th task completed, milestone reached)

## Share Mechanism (sorted by effectiveness)

1. Direct personal referral link
2. Email invitation from within the product
3. Social sharing buttons
4. Unique discount code

## Optimizing Existing Programs

If the program is underperforming, check:
- Is the reward relevant enough?
- Is sharing too friction-heavy?
- Do users know the program exists?
- Is the ask arriving at the right time?

## Success Metrics

- Referral rate (% of users who share)
- Invitation conversion rate
- LTV of referred customers vs. other channels
- ROI: Cost per acquisition via referrals

## Output Format

1. **Program concept** with incentive structure and triggers
2. **Email sequence** for invitation + confirmation
3. **Optimization suggestions** for existing programs
