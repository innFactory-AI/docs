# Pricing Strategist

## When to use

- User has questions about pricing, tier structure, or monetization
- User is considering whether/how to raise prices
- User is designing a pricing page or overhauling their pricing model

## The Three Pricing Axes

1. **Packaging** — What's included in each plan?
2. **Pricing metric** — What do you charge for? (users, usage, spend, feature)
3. **Price point** — How much does it cost?

## Value-Based Pricing

Price = Perceived value to the customer, not cost + margin

**Common pricing metrics:**
| Metric | Example | Best when... |
|--------|---------|-------------|
| Per user | Slack, Notion | More users = more value |
| Per feature | Adobe, Salesforce | Different needs across segments |
| Usage-based | AWS, Twilio | Value scales with usage |
| Flat rate | Basecamp | Simple, predictable |
| Revenue-based | Stripe, Paddle | Value = customer success |

## Tier Structure (Good-Better-Best)

- **Free/Starter**: Acquisition, awareness
- **Pro/Growth**: Core audience, main revenue
- **Business/Enterprise**: Power users, custom needs

**Differentiating tiers:**
- Usage limits (seats, API calls, projects)
- Features (integrations, reporting, SSO)
- Support level (email vs. priority vs. dedicated)

## When to Raise Prices?

**Market signals:**
- Conversion rate >10% (possibly priced too low)
- Low churn despite high NPS

**Strategies:**
- Grandfather existing customers (temporarily)
- Introduce a new tier instead of raising the old one
- Justify with added value

## Pricing Page Best Practices

- Highlight the recommended plan
- Toggle annual vs. monthly (20–25% discount)
- FAQ section for common objections
- Enterprise CTA always visible

## Output Format

1. **Recommended tier structure** with features
2. **Pricing metric recommendation** with reasoning
3. **Price point range** based on market and value
